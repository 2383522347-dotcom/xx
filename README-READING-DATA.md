# 英语阅读 · 外部数据扩展说明

## 目标规模

- **每等级文章总数不低于 1 万篇**（小学 / 初中 / 高中 / 大学 / 出国 / 雅思 / 托福 / 哈佛）
- **每篇文章的题目池不低于 8000 个**：同一篇文章每次测验只随机抽 **5 道题**，因此每次阅读的题目组合都不同
- **高中及以上**：**长文章、学术论文风格**，专业方向可为任意（理工、社科、医学等）；选材建议来自**国外权威学术论坛、开放获取期刊**（如 PLOS、arXiv 科普文、Nature Communications 开放阅读、各学会官网等）的合规、可商用或已获授权的文本
- 文章需**规范、优质**；题型与高考 / 雅思 / 托福一致：主旨大意、推理判断、词义猜测、细节理解等

## 每次测验题数

- 应用内**每次测验固定为 5 道题**（在 `app.js` 中 `READING_QUESTIONS_PER_QUIZ = 5`）
- 这 5 道题从**当前文章的题目池**中随机抽取，故同一篇文章多次阅读会得到不同题目组合

## 如何达到 1 万篇/等级 + 每篇 8000 题

应用支持从**外部 JSON**（或后续扩展为 API）加载阅读数据，优先于内置 `READING_ARTICLES`。

1. **在 `app.js` 中设置外部数据地址**（约第 582 行）：
   ```javascript
   var READING_DATA_URL = "reading-data.json";  // 同目录或同源下的 JSON
   // 或
   var READING_DATA_URL = "https://你的域名.com/data/reading.json";
   ```

2. **JSON 格式**（与 `data.js` 中 `READING_ARTICLES` 一致）：
   ```json
   {
     "小学": [
       {
         "title": "文章标题",
         "sentences": [
           { "en": "English sentence.", "cn": "中文句子。" }
         ],
         "questions": [
           { "q": "题目内容？", "options": ["A", "B", "C", "D"], "correct": 0 }
         ]
       }
     ],
     "初中": [ ... ],
     "高中": [ ... ],
     "大学": [ ... ],
     "出国": [ ... ],
     "雅思": [ ... ],
     "托福": [ ... ],
     "哈佛": [ ... ]
   }
   ```
   - `correct` 为正确选项下标：0=A，1=B，2=C，3=D
   - **每等级**可放置 **≥1 万篇文章**
   - **每篇文章**的 `questions` 数组可包含 **≥8000 道题**；应用每次测验从中**随机抽 5 道**

3. **高中及以上：长文与学术论文**
   - 选材：国外权威学术论坛、开放获取期刊、学会官网等**合规且可使用的**长文或论文摘要/综述（注意版权与使用许可）
   - 结构：多段落、有主题句与论据、学术用语；专业方向不限（科学、工程、社科、医学等）
   - 将长文按句或按段拆成 `sentences` 数组，每项 `{ en, cn }`；并为该文配备不少于 8000 道的题目池（主旨、推理、词义、细节等）

4. **题目类型建议**（与高考 / 雅思 / 托福一致）
   - 主旨大意（main idea）
   - 推理判断（inference）
   - 词义猜测（vocabulary / phrase）
   - 细节理解（detail）

5. **数据量建议**
   - 每等级 ≥1 万篇 × 每篇 ≥8000 题 → 需通过**外部 JSON 或 API + 数据库**提供，不宜全部写在 `data.js` 中
   - 若单文件过大，可拆成按等级或按批次的多个 JSON，并在 `app.js` 中扩展为按等级/分批请求（如 `READING_DATA_URL_BY_LEVEL["高中"] = "..."`）

6. **加载时机**  
   用户点击首页「英语阅读」时，应用会尝试请求 `READING_DATA_URL`；成功则使用外部数据，否则回退到 `data.js` 内置数据。

## 内置数据说明

当前 `data.js` 中 `READING_ARTICLES` 为**示例数据**：  
- 高中、大学各含一篇**长篇学术风格范文**（多段落、学术用语），其余为短篇示例  
- 每篇示例的题目池远小于 8000（仅作格式与题型示范）  
- 实际达到「每等级 ≥1 万篇、每篇 ≥8000 题」需使用上述**外部 JSON/API** 方案，并从权威学术论坛或开放获取期刊等渠道整理合规长文与题目。
